# 每日输出模板

每天 DECIDE 步骤产出，严格用这个结构，禁止偏离。这是 skill 的"强制 slot"——把结构固化在模板里，不靠每次临场记得，避免剧场化。

```
## 今日主攻 (Priority One)

**具体动作：** [具体到能立刻动手。反例："做CJC"。正例："改 CJC hero 区文案，把交易所 insider 背景放到第一屏"]

**为什么是它：** [命中三维度的哪一个 / 指向哪个瓶颈。一两句话。]

**完成判定：** [一个明确的终点。反例："优化转化"。正例："hero 区改完并部署上线，新文案肉眼可见"]

---

## 顺势推飞轮（≤2 个，可选）

- [小动作1]：只放能利用碎片时间、且反哺主攻的
- [小动作2]：

---

## 今日不做 / 推迟（附理由）

- [项目A]：[为什么今天不做] → 推迟到 [什么条件满足]
- [项目B]：...
```

## 填写纪律

- **Priority One 永远只有一个。** 如果忍不住想写两个，说明 ORIENT 的瓶颈判断没收敛，回去重新定位瓶颈。
- **"今日不做"不能省。** 这一段的作用是让被砍的想法带着理由离场，不再回来抢注意力。省了它，焦虑就回来了。
- **完成判定必须是可验证的终点**，不是"推进""优化""搞一下"这类没有终点的动词。
- 交付时不要附加冗长解释，模板本身就是交付物。
